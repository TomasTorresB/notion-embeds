On every page load / refresh
1
Check JS memory always empty after refresh
Access token is gone. This is expected — memory doesn't survive navigation. Move to next option.
2
Check for refresh token survives refresh
Stored in an HttpOnly cookie on your app's own domain (set by your BFF), or in a service worker. If found, POST it to the IdP's token endpoint directly — get a new access token in the response body. No redirect, no visible UI. Load time: ~100–300ms.
3
If no refresh token — try silent auth via IdP session invisible to user
Launch a hidden iframe (or background popup) pointing at the IdP with prompt=none. The IdP checks its own HttpOnly session cookie. If the user is still logged in at the IdP, it issues a new auth code silently — no password prompt. The SPA exchanges this for a fresh access token. The user sees nothing.
4
If IdP session also expired — full login
Only now does the user see the login screen. This is what happens after a long absence (days/weeks), or when the user explicitly logs out, or when the IdP enforces a maximum session lifetime.
