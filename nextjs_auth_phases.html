Phase 1 — Login initiation
User clicks "Sign in" → PKCE flow begins server-side
1
browser
React component calls fetch('/api/auth/login') — a plain HTTP request to your own Next.js server. No IdP involved yet.
2
server
Route Handler generates the code_verifier and code_challenge. Stores the verifier in a short-lived HttpOnly cookie called something like pkce_verifier. Returns a redirect URL pointing at the IdP.
3
browser
Navigates to the IdP login page. User authenticates. The pkce_verifier cookie travels with every request to your domain — but never to the IdP (different origin).
Phase 2 — Callback & token exchange
IdP redirects back → server exchanges code for tokens, sets cookies
4
IdP
Redirects browser to https://yourapp.com/api/auth/callback?code=X — your Next.js server route, not a client page.
5
server
Route Handler reads the code from the URL and the pkce_verifier from the incoming cookie. Makes a server-to-server POST /token request to the IdP — this never touches the browser at all.
6
server
Server receives the access_token and refresh_token in the response body. Neither token ever reaches the browser. Server stores them — typically in an encrypted session stored server-side (Redis, DB) or in an encrypted HttpOnly cookie.
7
server
Sets a session cookie — HttpOnly; Secure; SameSite=Lax — containing either the encrypted tokens or a session ID. Redirects browser to the app dashboard.
Phase 3 — Authenticated requests & token refresh
Browser never holds a token — server proxies everything
8
browser
React component calls fetch('/api/user') — again, your own Next.js server. The session cookie is automatically attached by the browser.
9
server
Route Handler reads the session cookie, decrypts it, checks if the access token is still valid. If expired, silently calls POST /token with the refresh token — entirely server-side. Updates the session cookie with the new tokens. Then proxies the request to your real API with the access token in the Authorization header.
10
browser
Receives the API response. Has no idea a token refresh happened. Has never seen an access token, a refresh token, or a client secret at any point.
